■ ネットワーク層
・IP（Internet Protocol）-TCP/IPの IP のこと。 インターネット通信を行うためのルール
　　　　　　　　　　　　　　　　OSI参照モデルでは、ネットワーク層で動作する。インターネット上でデータ通信をするためのプロトコル
　　　　　　　　　　　　　　　　 TCP/IPの階層モデルにおいてはインターネット層で動作するプロトコル

・IPヘッダ - IPパケットのヘッダ部にある。「IP」というお約束事に従って通信するために必要な情報や説明書き

・ICMP (Internet Control Message Protocol) -  IPプロトコルの「エラー通知」や「制御メッセージ」を転送するためのプロトコル
　　　　　　                                                    ICMPは、IPの代わりに通信の疎通状態や処理エラーを報告するといった、IPの補助をする。
   
・ICMPメッセージ - ICMPメッセージは「タイプ」「コード」「チェックサム」「データ」の4つのフィールドにより構成
　　　　　　　　　　　ICMPメッセージは大きく2種類あります問い合わせ(Query)とエラー通知（Error）
　　　　　　　　　　　あるノードから特定のノードに対する通信状態を確認することができます。パケットトレーサーのpingなど
　　　　　　　　　　　ノード間の通信で経路途中でパケットが廃棄されたりした場合に、その原因を送信元のノードにエラー通知する

・IPアドレス - デバイス(PCやスマホ)を識別する番号のこと
　　　　　　　　ネットワーク通信を行う場合に利用される、形式がIPv4（２進数と１０進数）とIPv6（１６進数）とある   (IPv4では、32ビット。IPv6では128ビット)

　　・オクテット - 1オクテット（octet）は、8ビットのこと　1バイト＝８ビット＝１オクテット
　　　　　　　　　　1バイト=8ビットというのは絶対ではないから、曖昧さを無くすために確実にオクテットが使用されることが多い

　　・ネットワーク部 - そのIPアドレスが属しているネットワークを識別するための部分 192.168.1.1の場合192.168.1がネットワーク部
　　・ホスト部 - ネットワーク内のコンピュータを識別するための部分　192.168.1.1の場合　最後の1がホスト部
　　・クラス - ネットワークを構成するパソコンの台数に応じて、３つのクラスに分かれます。 
                       大規模なネットワークならクラスＡ、中規模ならクラスＢ、小規模の場合はクラスＣ
　　・プライベートIPアドレス - 特定のネットワーク内で割り当てられるIPアドレスをプライベートIPアドレス（ローカルIPアドレス）
　　　　　　　　　　　　　　　LANやWAN内の特定内で割り当てられる端末ごとにある名札
　　・グローバルIPアドレス - ICANN（別名IANA）と呼ばれる国際的な機関が管理しており、国やプロバイダー（ISP）によってあらかじめ使用できる範囲が決められている
　　　　　　　　　　　　　　LANやWANが世界で使えるインターネットに接続して自分が何者か判断する名札

・ネットワークアドレス - Pアドレスにおいて、ホスト部のビットが全て0のIPアドレスで、ネットワークそのものを表すアドレス 
                                        ex:IP 192.168.0.1/24  　ネットワークアドレス 　　192.168.0.0　（サブネットで切って全て０にして求めた最小値、はじまりの数）
・ブロードキャストアドレス - ネットワーク内のすべての機器（ノード）にデータを一斉配信するために使われる特殊なアドレス
 　　　　　　　　　　　  ex:IP 192.168.0.1/24　ブロードキャストアドレス　192.168.0.255（サブネットで切って全て１で求めた最大値）
・ループバックアドレス - 現在使用しているコンピュータを指す特別なIPアドレス　自分の端末のIPアドレスのこと
                                       そもそもループバックとはある機器から送信されたデータや信号を、オウム返しにその機器へ送信し返す機器や仕組み。
　　　　　　　　　　　　　　　　　　　　　　　また、そのように自らへ戻っていく信号やデータの流れのこと
・サブネット - ネットワーク内のネットワークのこと。サブネットは、ネットワークをより効率的にして、ルーターを通らなくても短い距離で届けられるようになった。
・サブネットマスク - ネットワークの範囲を定義するために使用される。
　　　　　　　　　　IPアドレスのどこからどこまでがネットワーク情報（１の場所）で、どこからどこまでがコンピュータ情報（０の場所）を示す情報
                                  ex:198.51.100.xxx/255.255.255.0    11111111.11111111.11111111.00000000 となる
　　・プレフィックス表記 - 198.51.100.xxx/24」のようにIPアドレスとサブネットマスクを同時に表現する書き方のこと 
　　　　　　　　　  ex:198.51.100.xxx/255.255.255.0　⇨ 198.51.100.xxx/24


　　　2進数：11111111.11111111.11111111.00000000
　　　10進数表記：255.255.255.0
　　　プレフィックス表記：1が左から24個あるから　/24
　　　というように表している


・デフォルトゲートウェイ - LANからインターネットなどの、外部ネットワークに接続する際の通信の出入口の役割を果たす仕組みでルーターとかに備わっている仕組み

・IPv6(Internet Protocol Version 6) - インターネットプロトコル(IP)の１つ。
　　　　　　　　　　　　　　　　　　　　　　　元々は「IPv4」が主流でしたが、インターネットが急速に普及したことでIPアドレスが枯渇する恐れがあるとして、新しく「IPv6」が誕生しました
　　　　　　　　　　　　　　　　　　　　　　　IPv4（約43億個）とIPv6（約340澗（かん）個）では「使用できるIPアドレスの数」が異なります。
　　・サブネットプレフィックス - サブネットマスクをプレフィックス表記したもの、IPアドレスの中でネットワークアドレスを示す前のほうの部分を指します
　　・インターフェイスID - 64ビットあり、リンク上でホストを識別するために使用されます.「EUI-64」と「匿名」という仕組みで手動でも、自動でも組める
 


・ルーター - ネットワーク間の接続をおこなう機器の1つ。複数の異なるネットワークの間に存在し、データを中継する役割。
　　・コンソール接続 - コンピュータさんと意志疎通するときに使う道具　RJ-45など

　　・VTY接続（virtual teletype） - ルータなどの通信機器にネットワークを通じて別のコンピュータから接続し、操作や管理を行うための仮想的な接続口
　　・AUX接続（Audio Return Channel）- ARCに対応している機器同士であれば、HDMIケーブル1本でつなぐだけで、双方のデジタル音声を楽しむことができます。
　　　　　　　　　　　　　　　　　　　　　　一方的だった信号を双方向になり、テレビ側とオーディオ側の両方ともがARC対応のHDMI端子を搭載していなければならない。
　　　　　　　　　　　　　　　　　　　　　　最高192kHz/24bitのLPCMまで、もしくは5.1chまでのフォーマットのみに限られる。
 
　　・ルーティングテーブル - ルーター内で経路情報を指し示す経路表のこと。
　　　　　　　　　　　　　　 　ルーターを通過するデータのヘッダー情報と照らし合わせて、
　　　　　　　　　　　　　　　「このパケットを正しく届けるために次はどのルーターにデータを引き渡すと効率的か」を記載してあるテーブルのこと
　　　　・ネクストホップ - ルータなどが持つ経路表（ルーティングテーブル）に記載される情報の一つで、個々の宛先ネットワーク毎に決められた、次に転送すべき隣接ルータ（のIPアドレス）のこと

　　・ARPテーブル - ARP（Address Resolution Protocol）によって割り出したIPアドレスとMACアドレス（EthernetやWi-Fiなどの物理アドレス）の対応表
　　　　・ARP - ARP（Address Resolution Protocol）は、IPアドレスからネットワーク上のMACアドレスを求めるためのプロトコル

　　・コアルータ - 通信キャリアやISPネットワークの通信を中継するために利用されるルータでより大量のデータ量を高速に送れる、高速道路のような感じ
　　　　　　　　　　基幹ネットワークのバックボーンを構成するルータで、高速なパケット転送を実現する。大型のルータが適用される
　　・センタールータ - 企業の拠点間WANネットワークで本社拠点・データセンタ等、企業で中心的に利用されるネットワークを接続するために利用される
　　　　　　　　　　　　エッジルータより優秀だからネットワークトラフィック（一定のデータ通信量）が多く集中する必要があるときはこちらが選ばれる
　　・エッジルータ - 企業の拠点間WANネットワークで支社や営業所等の拠点ネットワークを接続するために利用される
　　　　　　　　　　　センタールーターに比べてネットワークトラフィックが集中しない、支社や営業所等の通信量が少ない拠点についてはエッジルータが選ばれる　　　　　　　
　　・ブロードバンドルータ - 家庭環境やSOHO環境でインターネット回線に接続する目的で利用される
　　　　　　　　　　　　　　　主にインターネット回線に接続する目的で利用されます。インターネットVPNの機能を持ったブロードバンドルータでは企業内ネットワークに接続する事も可能
 

・ルーティング - ネットワーク上でデータを転送する際、その経路を導き出すこと。「どっちに行けば目的地に着けるかな～」を判断する仕組み。要はルート決め

　　・直接接続 - 国内の携帯電話事業者とSMS配信事業者が直接接続すること　メリットは 品質・価格・障害対応の速さ
　　・関節接続 - SMS配信事業者が別のSMS配信事業者のプラットフォームを介して間接的にSMSを送信すること
　　　　　　　　　そのため、直接接続と比べてSMSを送信した先の携帯電話事業者と直接連絡を取ることができず、品質の低下や障害が発生する可能性があり

　　・スタティックルーティング - 別名・静的ルーティングは、予めネットワーク管理者が設定したルーティングテーブルに則ってルーティングを行う
　　・ダイナミックルーティング - ルーター同士が常にルート情報を交換しあい、ルーティングテーブルの最適化を動的に行う

　　　　　・ディスタンスベクタ型 - ルータ同士でネットワークアドレス/サブネットマスク+メトリックの情報を交換　ネットワーク構成に何も変更がなくても定期的にルート情報を交換する
　　　　　・リンクステート型 - ルータは自分が持っているインターフェイスのリンクステート情報のみを交換　
　　　　　　　　　　　　　　　　インタフェースの情報をリンクステート情報(Link State Advertisement : LSA)を集めてリンクステートデータベース(Link State Data Base : LSDB)を作成
　　　　　　　　　　　　　　　　SDB内のルータ自身を起点として、リモートネットワークへの最小コストとなるルートを最適ルートとしてルーティングテーブルに登録します。
　　　　　・ハイブリッド型(拡張ディスタンスベクタ) - Ciscoが独自にディスタンスベクタ型を拡張した物なのでベースはディスタンスペクタ型に近い
　　　　　　　　　　　　　　　　　　　　　　　　　　　違うのは、ルーター同士の情報交換は常にではなくネットワークに変更があったときのみ
　　　　　　　　　　　　　　　　　　　　　　　　　　　交換したルート情報からDUAL(Diffusing Update ALgorithm)によって、
　　　　　　　　　　　　　　　　　　　　　　　　　　　最適ルートを決定し、最適ルートだけでなく利用可能なバックアップルートもあらかじめ決めておく




■ トランスポート層


・コネクション型通信 - データ通信において、通信を開始する前に相手との間で仮想的な専用通信路（コネクション）を確立し、それを通じてデータの送受信を行う通信方式。
                    送信したデータが相手に確実に届き、データの到着順も送信順通りになることが保証される
　　・ACK（nowledgement） - データ通信において受信側から送信側に送られる、データを正常に受信したことを示す信号

・コネクションレス型通信（データグラム方式） - 事前の接続確認を行わずいきなりデータ転送を始める通信方法のこと
　　　　　　　　　　　　　　　　　　　　　　　　コネクション型通信とは異なり、事前にコネクションの確立を行いません
　　・TCP(Transmission Control Protocol) - インターネットの主要なプロトコルの1つで、1対1のセッションによる信頼性の高い通信を行うためのプロトコル
　　　　　　　　　　　　　　　　　　　　　　　　　（確実性をもった通信を行いたい場面で利用される）　　　　
　　・スリーウェイハンドシェイク - データ転送を行う前にコネクションの確立を行います.TCPにおいて使用されるコネクションの確立のこと 　“3回のやりとり" によって確立される

　　・Telnet - ネットワークを介して２つのホスト間で通信を行うためのプロトコル　
　　　　　　　　遠隔地にあるホストを、仮想端末ソフトを利用して操作することができる
　　・HTTP(HyperText Transfer Protocol) - データ通信の際に通信内容を暗号化しない方式です。
　　　　　　　　　　　　　　　　　　　　　　　　http通信を使ったWebサイトでデータの入力や発信を行なうと第三者に通信内容を知られてしまう可能性あり

 　　・FTP（File Transfer Protocol）　 - ファイルを転送するための通信規格です。 クライアントとサーバ間で、ファイルのアップロードやダウンロードを行うときに使われるプロトコルです。
　　・SMTP(Simple Mail Transfer Protocol) - インターネットなどのTCP / IPネットワークで標準的に用いられる、メールを送信するための通信プロトコルの一つ

・TCPヘッダ - TCPでのデータの伝送単位を「TCPセグメント」（TCP segment）あるいは「TCPパケット」と呼び、その先頭にある制御情報が記された部分のことをTCPヘッダという。 
　　　　　　　　内容が固定されている部分が先頭から20バイト、オプションが4バイト単位の可変長となっている。

　　・UDP(User Datagram Protocol) - インターネット上で、動画の再生や、DNSのルックアップなど、特に時間を重視した通信に使用される通信プロトコル
　　・RTP(Real-time Transport Protocol) - インターネット上の音声や映像配信のための標準パケット形式の規格

　　・SNMP（Simple Network Management Protocol） - TCP/IPネットワーク環境の監視・管理を担う、プロトコルのひとつです。 
　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　ルータやスイッチングハブといったネットワーク機器、Windows、UNIXといったサーバなどの状態を監視をする。
　　・DHCP(Dynamic Host Configuration Protocol) - コンピュータがネットワークに接続して通信する際に必要な設定情報を自動的に割り当てる機能
　　・TFTP(Trivial File Transfer Protocol) - ファイル転送に使用する UDP プロトコルです。 TFTP は、リモート・サーバーとの間でファイルの読み取りや書き込みをすることができます 
　　・RIP（Routing Information Protocol） - IPネットワークで用いられるルーティングプロトコルの一つ。
　　　　　　　　　　　　　　　　　　　　　　　　　ルータなどの通信機器の間で経路情報を交換し、ある地点から別の地点までの最短経路を割り出すのに使われる

・UDPヘッダ - UDPデータグラムの先頭部分にある、制御情報などを一定の形式で記述した領域。 この後ろに送りたいデータ本体（ペイロード）を加えて、一つのUDPデータグラムとなる
